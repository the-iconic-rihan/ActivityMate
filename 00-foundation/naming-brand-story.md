# Naming Brand Story

## Overview

"ActivityMate" is the working name used throughout this repository and in [PRD-001](../PRD-001_Master_Product_Requirements.md). It is explicitly a placeholder, not a final brand decision — final naming, logo, and verbal identity are owned by [../33-branding-identity/](../33-branding-identity/README.md) and should be resolved before any paid marketing spend in [go-to-market-strategy.md](../01-business-strategy/go-to-market-strategy.md) locks in a name across app store listings and ad creative.

## Context / Problem

A name for this category carries more risk than it does for most consumer apps, because the category itself is ambiguous by default. Any name that reads as even slightly dating-coded will (a) trigger App Store / Play Store category mis-classification risk, (b) repel the safety-conscious segment of the target persona set, particularly women aged 25-34, whose retention is a named success metric in PRD-001 Section 10, and (c) contradict the explicit Non-Goal in PRD-001 that this is not a matrimony, hookup, or anonymous chat platform. At the same time, the name needs to signal energy and spontaneity — "let's go do something" — not sound like enterprise software or a safety-compliance product.

## Naming Criteria

Four criteria were set before any candidate names were generated:

1. **Trust-signaling.** The name should not require explanation to reassure a first-time user's parent or roommate that this is not a dating app.
2. **Non-dating-coded.** No romantic, matching, or swipe-adjacent vocabulary ("match," "spark," "flame," "date").
3. **Code-switching fluency.** The target Mumbai persona set moves fluidly between Hindi and English in speech; the name should be pronounceable and not accidentally carry an unintended meaning in Hindi, Marathi, or common code-switched slang.
4. **Availability.** Domain (.com or .in), Play Store / App Store handle, and Instagram handle available or acquirable without significant premium cost, since early growth in [go-to-market-strategy.md](../01-business-strategy/go-to-market-strategy.md) depends on a clean, memorable social handle.

## Options Considered

| Candidate | Read | Verdict |
|---|---|---|
| ActivityMate | Descriptive, safe, slightly generic/corporate | Adopted as placeholder — clears criteria 1 and 2 cleanly, weakest on distinctiveness |
| Tolly (working shortlist name, "let's do it together") | Playful, short, ambiguous meaning without context | Shortlisted for branding-identity evaluation |
| Sanga (from "sangha," Sanskrit/Hindi for community/fellowship) | Culturally resonant, community-coded rather than activity-coded | Shortlisted; risk of skewing toward spiritual/wellness category association |
| PlanB / Plann | Wordplay on "plan," English-only pun | Rejected — pun does not translate across code-switching, and "Plan B" carries an unrelated negative connotation (contraception brand recognition in some markets) |
| Meetstr / Huddle-style portmanteau | Startup-generic naming pattern | Rejected — sounds like a B2B SaaS tool, undermines the consumer, spontaneous-energy brand feel |

## Recommendation & Rationale

Retain **ActivityMate** as the working name through MVP build and Mumbai soft launch, since renaming mid-build has near-zero cost right now (no installed base, no brand equity yet) but material cost later (App Store listing history, word-of-mouth referral loss, re-issuing verified badges/certificates). Route final resolution through [../33-branding-identity/](../33-branding-identity/README.md), which owns qualitative name-testing with real Mumbai target users — specifically testing whether a name reads as safe to a first-time female user's family, since that reaction is a stronger predictor of the trust-signaling criterion than founder intuition.

The naming decision is deliberately sequenced *after* [company-strategy-thesis.md](company-strategy-thesis.md) and *before* heavy paid growth spend — branding should express the activity-first, trust-first thesis, not the other way around.

## Risks & Open Questions

- Domain and handle availability for shortlisted alternatives has not yet been verified — action item for branding-identity.
- "ActivityMate" as a permanent name risks sounding like a scheduling utility rather than a social product; this is an acceptable tradeoff for a placeholder but not for a final brand.
- Open question: does the final name need to work as a standalone word in Hindi/Marathi advertising copy, or is Hinglish code-switched usage ("chal ActivityMate pe activity dhoondte hain") acceptable — pending qualitative testing.
- Trademark clearance in India (Class 42/45 for the app, relevant classes for any physical event branding) has not been run and must precede final adoption.
